# STOPS

In-house development of the STOPS model at SANDAG

## Directory Structure

### `/docs/`

All internal documentation that explains:

- Project workflow
- STOPS setup
- Assumptions
- File requirements

### `/scripts/`

All scripts used for:

- Preprocessing inputs
- Producing STOPS-format files
- QA/QC automation
- Output postprocessing

### `/configs/`

Each folder holds STOPS parameters, scenario-specific settings, and config files.

### `/scenarios/`

Dedicated folders for each scenario, with:

- Inputs (linked or documented)
- Run notes
- Outputs (or links to SharePoint outputs)

### `/qa_qc/`

Validation scripts, logs, and automated checks.

### `/archive/`

Old configs or scripts no longer used but preserved for reference.
